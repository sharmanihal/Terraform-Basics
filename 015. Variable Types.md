
### **1. String**

A single line of text.

```hcl
variable "env" {
  type    = string
  default = "dev"
}

output "env_name" {
  value = var.env
}
```

➡️ Output: `dev`

---

### **2. Number**

A numeric value (integer or float).

```hcl
variable "instance_count" {
  type    = number
  default = 3
}

output "count" {
  value = var.instance_count + 1
}
```

➡️ Output: `4`

---

### **3. List (ordered collection, index-based)**

```hcl
variable "zones" {
  type    = list(string)
  default = ["us-east-1a", "us-east-1b", "us-east-1c"]
}

output "first_zone" {
  value = var.zones[0]
}
```

➡️ Output: `us-east-1a`

✅ **Use case:** when order matters (e.g., availability zones).

---

### **4. Map (key-value pairs)**

```hcl
variable "instance_amis" {
  type = map(string)
  default = {
    dev  = "ami-123"
    prod = "ami-456"
  }
}

output "prod_ami" {
  value = var.instance_amis["prod"]
}
```

➡️ Output: `ami-456`

✅ **Use case:** easily fetch a value using a key.

---

### **5. Object (structured data)**

Like a record — multiple named attributes with fixed types.

```hcl
variable "app_config" {
  type = object({
    name = string
    port = number
  })
  default = {
    name = "web"
    port = 8080
  }
}

output "app_name" {
  value = var.app_config.name
}
```

➡️ Output: `web`

---

### **6. Boolean**

True/False values — often used in conditions.

```hcl
variable "enable_logs" {
  type    = bool
  default = true
}

output "logs_enabled" {
  value = var.enable_logs ? "Enabled" : "Disabled"
}
```

➡️ Output: `Enabled`

---

### **7. Set (unordered unique values)**

Like a list, but **no duplicates** and **order doesn’t matter**.

```hcl
variable "unique_zones" {
  type    = set(string)
  default = ["us-east-1a", "us-east-1a", "us-east-1b"]
}

output "zones" {
  value = var.unique_zones
}
```

➡️ Output: `["us-east-1a", "us-east-1b"]`

---

### **8. Tuple (ordered collection, mixed types allowed)**

```hcl
variable "server_info" {
  type    = tuple([string, number, bool])
  default = ["web", 8080, true]
}

output "server_port" {
  value = var.server_info[1]
}
```

➡️ Output: `8080`

✅ **Use case:** when you want ordered but mixed-type data.

---

### **9. Any**

Accepts any type of value — string, number, list, etc.

```hcl
variable "flexible" {
  type    = any
  default = [1, 2, "three"]
}

output "flexible_value" {
  value = var.flexible
}
```

➡️ Output: `[1, 2, "three"]`
