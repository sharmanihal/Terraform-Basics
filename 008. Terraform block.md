### **Terraform Block**

#### **1. Purpose**

The `terraform` block is used to configure **Terraform’s own settings**, not cloud resources.
It defines things like:

1. Required Terraform version
2. Required providers and their sources
3. Backend configuration (for state storage)

---

#### **2. Example**

```hcl
terraform {
  required_version = ">= 0.12.0"

  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 3.27"
    }
  }
}
```

---

#### **3. Explanation**

| Setting              | Description                                                                                                                      |
| -------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| `required_version`   | Ensures Terraform CLI version meets a minimum version requirement. Prevents older, incompatible versions from running your code. |
| `required_providers` | Declares which providers your configuration depends on (e.g., AWS, Google, Azure).                                               |
| `source`             | Tells Terraform where to download the provider from. Usually `"hashicorp/<provider>"`.                                           |
| `version`            | Specifies which version (or version range) of the provider to use. Helps prevent breaking changes.                               |

---

#### **4. Example Meaning**

* Terraform must be **version 0.12.0 or higher**.
* It will download the **AWS provider** from **HashiCorp’s registry**.
* The provider version should be **3.27 or compatible newer versions** (due to `~>` operator).
