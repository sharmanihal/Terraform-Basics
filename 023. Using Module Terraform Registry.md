
### **What is Terraform Registry?**

The **Terraform Registry** is a public (and sometimes private) repository of reusable Terraform modules shared by the community or official providers.
It helps you **reuse existing infrastructure code** instead of writing it from scratch.

---

### **Why Use It?**

* Saves time â€” prebuilt, tested modules for common infra (VPCs, EC2, S3, etc.)
* Ensures best practices (official modules follow AWS/Azure/GCP standards)
* Makes code DRY and maintainable

### **Example: Using AWS VPC Module from Terraform Registry**

You can find modules on:
ðŸ‘‰ [https://registry.terraform.io](https://registry.terraform.io)

Example: [AWS VPC Module](https://registry.terraform.io/modules/terraform-aws-modules/vpc/aws/latest)

```hcl
module "vpc" {
  source  = "terraform-aws-modules/vpc/aws"
  version = "5.5.0"

  name = "my-vpc"
  cidr = "10.0.0.0/16"

  azs             = ["us-east-1a", "us-east-1b"]
  public_subnets  = ["10.0.1.0/24", "10.0.2.0/24"]
  private_subnets = ["10.0.3.0/24", "10.0.4.0/24"]

  enable_nat_gateway = true
  tags = {
    Environment = "dev"
  }
}
```

### **Steps to Use a Registry Module**

1. **Search Module**

   * Go to [Terraform Registry](https://registry.terraform.io)
   * Choose provider (e.g., AWS, Azure, GCP)
   * Copy the moduleâ€™s `source` and optional `version`

2. **Add to Your Code**

   * Paste the module block in your Terraform file (`main.tf`)

3. **Initialize**

   ```bash
   terraform init
   ```

   * Downloads the module from the registry.

4. **Plan and Apply**

   ```bash
   terraform plan
   terraform apply
   ```

---

### **Module Versioning**

Always pin a version to avoid future breaking changes:

```hcl
version = "5.5.0"
```

---

### **Module Sources Types**

| Type         | Example                                                     | Description             |
| ------------ | ----------------------------------------------------------- | ----------------------- |
| **Local**    | `source = "./modules/s3-module"`                            | From local folder       |
| **Registry** | `source = "terraform-aws-modules/vpc/aws"`                  | From Terraform Registry |
| **GitHub**   | `source = "github.com/user/repo//path?ref=tag"`             | From a Git repo         |
| **S3/GCS**   | `source = "s3::https://s3.amazonaws.com/bucket/module.zip"` | From cloud storage      |

---

### **Best Practices**

* Always specify **module version**
* Use **variables.tf** to pass module inputs clearly
* Store commonly used modules in your own **private registry or repo**
* Avoid editing downloaded modules directly â€” override via variables instead
