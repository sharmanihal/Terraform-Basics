
### **`terraform.tfvars` File**

When you have **many variables** or don’t want to hardcode defaults, you can store their values in a **`.tfvars` file**.
Terraform automatically loads this file during `plan` or `apply`.

---

#### **Example**

**`variables.tf`**

```hcl
variable "region" {
  description = "AWS region to deploy resources"
  type        = string
}
```

**`terraform.tfvars`**

```hcl
region = "us-west-1"
```

Now, when you run:

```bash
terraform plan
```

Terraform automatically picks up the value from `terraform.tfvars`.

---

#### **Custom .tfvars file**

You can also specify a different file name:

```bash
terraform plan -var-file="dev.tfvars"
```

---

✅ **Key Points:**

* `terraform.tfvars` is auto-loaded if present.
* Useful for managing multiple environments (e.g., `dev.tfvars`, `prod.tfvars`).
* Keeps sensitive or environment-specific data separate from code.
