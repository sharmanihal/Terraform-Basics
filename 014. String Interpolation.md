
### **String Interpolation in Terraform**

**String interpolation** lets you embed variables or expressions directly inside strings.
It’s useful for creating dynamic values (like names, tags, or resource properties).

---

#### **Syntax**

```hcl
"${var.variable_name}"
```

Or (in newer Terraform versions) simply:

```hcl
"prefix-${var.variable_name}"
```

---

#### **Example**

```hcl
variable "app_tag_name" {
  type    = string
  default = "app"
}

resource "aws_instance" "app_server" {
  ami           = "ami-0abcdef1234567890"
  instance_type = "t2.micro"

  tags = {
    Name = "ec2-${var.app_tag_name}"
  }
}

resource "aws_security_group" "app_sg" {
  name = "sg-${var.app_tag_name}"
}
```

---

✅ **Result:**

* EC2 tag → `ec2-app`
* Security group name → `sg-app`

---

**Note:**
You can also use interpolation for arithmetic or logical expressions:

```hcl
"${count.index + 1}"
"${var.env == "prod" ? "t3.large" : "t3.micro"}"
```
